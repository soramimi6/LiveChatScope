# LiveChatScope — UI/UX 改修メモ

> **記録日**: 2026-06-21  
> **出典**: 初回ローカル環境での実画面確認（ユーザー feedback）  
> **優先度**: Phase B / UI polish（第一弾完成後）

---

## 背景

第一弾プロトタイプを Windows ローカルで起動し、実際の YouTube URL で取得〜進捗画面まで動作確認した際の気づき。  
機能は問題なく動作しているが、**進捗画面の体感**に改善余地あり。

---

## 改修案一覧

### UX-01: 進捗画面のステップ表示を詳細化

| 項目 | 内容 |
|------|------|
| **現状** | 1 行の `stepLabel` のみ（例: 「チャット取得中…」「分析処理中…」）。プログレスバーも fetch 状態に粗く紐づく（pending=10%, fetching=40%, fetched=80%） |
| **要望** | ユーザーが「今何をしているか」を段階的に把握できるよう、ステップをもう少し詳細に表示したい |
| **案** | 取得 / 分析を分けた **ステップリスト**（チェックマーク付き）例: ① URL 登録 → ② チャット取得 → ③ 基本分析 → ④ 話題・盛り上がり分析 → ⑤ 完了 |
| **API 活用** | `GET /status` の `fetch_status`, `analysis_status`, `progress.analysis_stage`, `progress.analysis_stage_label` を表示に反映 |
| **触るファイル（想定）** | `frontend/app/analyze/[videoId]/page.tsx`, 必要なら `docs/ui-spec.md` §進捗画面 |
| **備考** | 現行 UI 文言に「Pipeline は後続タスクで拡張」と残っており、実装済み Pipeline と乖離。合わせて修正 |

---

### UX-02: ラベル文言「取得済みメッセージ」→「取得済みチャットコメント」

| 項目 | 内容 |
|------|------|
| **現状** | 進捗画面: `取得済みメッセージ: {n}`（`frontend/app/analyze/[videoId]/page.tsx` L88–89） |
| **要望** | **取得済みチャットコメント** に変更（プロダクト用語「ライブチャット」と揃える） |
| **案** | 進捗画面の表示文言のみ変更。API フィールド名 `messages_fetched` は互換のため維持可 |
| **触るファイル（想定）** | `frontend/app/analyze/[videoId]/page.tsx`, 必要に応じ `docs/ui-spec.md` |
| **備考** | 他画面（サマリー KPI 等）の「メッセージ」表記は別途、用語統一の観点で横断レビュー推奨 |

---

### UX-03: 取得件数の更新頻度を上げ、「動いている」感を強化

| 項目 | 内容 |
|------|------|
| **現状** | Backend が **500 件ごと** に DB 更新（`fetch_worker.py` の `batch_size = 500`）。Frontend は **2.5 秒ごと** にポーリング（`page.tsx` `setInterval(poll, 2500)`）。そのため画面上の数字が **500 件刻み** でしか増えず、待ち時間中のフィードバックが弱い |
| **要望** | もう少し高頻度で数字が動き、「処理中」であることを実感できるようにしたい |
| **案 A（Backend）** | 進捗更新間隔を短縮 — 例: `batch_size` を 500→100、または `_update_fetch_progress` を N 件 / M 秒ごとに追加呼び出し |
| **案 B（Frontend）** | ポーリング間隔を 2500ms→1000ms 程度に短縮（Backend 変更なしでも体感的に追従しやすくなるが、500 刻みは残る） |
| **案 C（UI）** | 件数横に **スピナー / パルスアニメ**、「最終更新: n 秒前」、取得レート（件/秒）の表示 |
| **推奨** | **A + B** の組み合わせ（体験改善が大きい）。大規模配信では DB 書き込み頻度とトレードオフ → Phase B で性能確認 |
| **触るファイル（想定）** | `backend/app/services/fetch_worker.py`, `frontend/app/analyze/[videoId]/page.tsx` |
