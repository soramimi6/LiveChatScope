# LiveChatScope — プロダクト概要

## 名称

**LiveChatScope** — ライブチャットを「覗く・分析する」スコープ（観測装置）のイメージ。

## 定位

> 配信後のチャットを、**個人配信者の振り返り資料**に変える Web ツール

YouTube Analytics の代替ではなく、**チャットリプレイに基づく配信運営の振り返り**に特化する。

## ターゲット

- **主**: 個人配信者（配信後の改善・スパチャお礼・切り抜き候補の抽出）
- **副**: 協力者（編集者・モデレーター）— エクスポート・ジャンプリンクの共有
- **想定外（主眼）**: 企業・MCN 等、既に高度な分析基盤を持つ組織

## 背景

YouTube ライブ配信のチャットリプレイは、配信者にとって重要な振り返りデータだが、
終了後に公式 API だけで全件取得することはできない。
また、CSV を落とすだけでは「どの区間が盛り上がったか」「次回何を改善するか」には直結しにくい。

LiveChatScope は取得から**話題俯瞰・盛り上がり検出・配信者向け実務出力**までを一つの Web ツールで提供する。

## 提供価値（配信者向け）

| 課題 | LiveChatScope の答え |
|------|----------------------|
| どこが盛り上がった？ | 盛り上がり候補＋ジャンプ URL |
| 配信の構成はどう流れた？ | 話題ブロック・構成タイムライン |
| スパチャのお礼が面倒 | お礼リスト（時刻・金額・ジャンプ付き） |
| 切り抜きどこから？ | 切り抜き候補パック（タイムスタンプ形式） |
| 間延びした区間は？ | 低活動区間の可視化 |

## フェーズ

### Phase A（MVP）

- 配信終了後、YouTube URL からチャットリプレイを取得
- 基本分析（密度・Top N・スパチャ）・検索・ジャンプ・エクスポート
- 個人利用・動作検証

### Phase A+（差別化 MVP）

- 配信サマリー、盛り上がり候補、話題ブロック、キーワード分析
- スパチャお礼リスト、切り抜き候補、低活動区間
- 個人配信者向け UI（サマリー / 話題 / 盛り上がり / 収益 / コミュニティ）

### Phase B（本番品質）

- ジョブ進捗・中断再開
- エラーハンドリング強化
- 大規模配信（10万コメント級）の性能確認

### Phase D（公開サービス・将来）

- ユーザー認証・レート制限
- 同一動画のキャッシュ共有
- 利用規約・プライバシー対応

## 技術方針（概要）

- **取得**: chat-downloader（Python）でリプレイチャットを取得
- **分析**: `time_in_seconds` を軸に時系列集計。形態素解析でキーワード・話題ブロック
- **ジャンプ**: `https://www.youtube.com/watch?v={VIDEO_ID}&t={秒}s`
- **Web**: フロント + API サーバー + DB（詳細は [アーキテクチャ.md](アーキテクチャ.md)）

## 制約・リスク

- 配信者がチャットリプレイを無効にしている場合は取得不可
- 非公式取得手段に依存するため、YouTube 仕様変更で障害の可能性あり
- 話題分析は「チャット上の推定話題」であり、配信内容と一致しない場合がある
- 公開サービス化時は ToS・プライバシー・ストレージコストの検討が必要
