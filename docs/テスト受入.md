## 1. 概要

第一弾（Phase A + A+ プロトタイプ）の **受入判定基準** を定義する。

| 項目 | 方針 |
|------|------|
| 第一弾 | **手動テスト中心** + 重要 API の自動テスト（実装フェーズで追加） |
| 合格基準 | 本ドキュメントの **Must** 項目をすべて Pass |
| 性能 | 10 万コメント級で **実用速度**（§6） |
| サンプル URL | 固定 URL に依存せず **取得手順** で管理（§7） |

### 1.1 第一弾完成の定義

以下をすべて満たした時点で `dev` → `master` マージ可:

1. Phase A Must 全 Pass
2. Phase A+ Must 全 Pass
3. E2E シナリオ（§5）1 回以上 Pass
4. 性能目標（§6）を 1 配信以上で確認
5. 既知の制限（§8）を README または UI に明記

## 5. E2E シナリオ（第一弾必須）

### E2E-01: 標準フロー（Must）

```
1. トップで YouTube URL 入力
2. 進捗画面で取得・分析完了を待つ
3. サマリータブ: KPI・話題 preview を確認
4. 盛り上がりタブ: 候補 1 件のジャンプを YouTube で確認
5. 収益タブ: スパチャ表示 or Empty
6. 詳細検索: キーワード 1 件ヒット
7. JSON / CSV エクスポート成功
8. 切り抜き Markdown / お礼 Markdown 取得（スパチャあり配信）
```

**合格**: 上記 8 ステップすべて成功。

### E2E-02: 再分析（Should）

```
1. 同一 video_id で再分析実行
2. 派生データが上書きされ、UI が最新結果を表示
```

## 6. 性能目標（10 万コメント級）

### 6.1 対象

| 区分 | 目標 |
|------|------|
| メッセージ数 | **≥ 50,000**（理想 100,000） |
| 配信時間 | 2 時間前後を推奨 |

10 万未満でも **5 万超** で代替可（第一弾 POC）。

### 6.2 計測項目

| ID | 項目 | 目標（目安） | 計測方法 |
|----|------|--------------|----------|
| P-01 | 取得 + 分析完了時間 | **≤ 30 分**（10 万 msg・ローカル PC） | ログ `created_at`〜`analyzed_at` |
| P-02 | サマリー API | **≤ 2 秒** | DevTools Network |
| P-03 | 密度 API | **≤ 2 秒** | 同上 |
| P-04 | メッセージ検索（初回） | **≤ 5 秒** | 同上 |
| P-05 | サマリータブ初期表示 | **≤ 3 秒**（API 完了後） | 体感 + Performance |
| P-06 | DB ファイルサイズ | **≤ 500 MB**（tokens 削除後） | ファイルサイズ |

未達時: ボトルネック記録 → Phase B で改善。第一弾は **P-02〜P-05 が Must**。

### 6.3 性能テスト手順

1. 5 万〜10 万コメントの配信を 1 本選定（§7）
2. クリーン DB または新 video でフル分析
3. P-01〜P-06 を記録（`docs/test-runs/` に結果メモ可・任意）
4. Must 項目が Pass なら性能合格

## 7. 手動テスト用サンプル URL 方針

### 7.1 原則

- **リポジトリに固定 URL をコミットしない**（配信の公開状態が変わるため）
- テスト実行者が **都度選定** し、ローカルメモまたは `docs/test-runs/`（gitignore）に記録

### 7.2 選定基準

| 用途 | 条件 |
|------|------|
| **標準 E2E** | チャットリプレイ有効、配信終了済み、メッセージ 1,000 件以上 |
| **スパチャテスト** | スパチャ 1 件以上含む |
| **性能テスト** | メッセージ 50,000 件以上、リプレイ有効 |
| **ネガティブ** | リプレイ無効 / 非公開（可能なら） |

### 7.3 記録テンプレート（`docs/test-runs/YYYYMMDD.md`・任意）

```markdown
## テスト実行 YYYY-MM-DD
- 実行者:
- 環境: Windows / Chrome
- URL-標準: (video_id のみ記載可)
- URL-性能: 
- 結果: E2E-01 Pass/Fail, P-02: 1.2s, ...
```

`docs/test-runs/` は `.gitignore` 推奨。

## 8. 既知の制限（UI / README に明記）

| 制限 | 説明 |
|------|------|
| 非公式取得 | chat-downloader 依存。突然動作しなくなる可能性 |
| 推定話題 | 配信内容とチャット話題は一致しない場合がある |
| 認証なし POC | 公開ネットワークへの無防備デプロイ不可 |
| スマホ | 閲覧程度。最適化なし |
| 英語 UI | 第一弾対象外 |
| Analytics | 同時視聴者数等は提供しない |

| 種別 | 対象 | ツール案 |

| E2E | E2E-01 主要路径 | Playwright（任意・Phase B） |
