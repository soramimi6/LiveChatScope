## 3. 開発環境

### セットアップ

```powershell
git clone https://github.com/soramimi6/LiveChatScope.git
cd LiveChatScope
git checkout dev && git pull origin dev

# Backend
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt -r requirements-dev.txt
uvicorn app.main:app --reload --port 8000

# Frontend（別ターミナル）
cd frontend
copy .env.example .env.local
npm install
npm run dev
```

Linux/macOS: `source backend/.venv/bin/activate`

- Frontend: http://localhost:3000
- API: http://localhost:8000/docs
- DB: `backend/data/livechatscope.db`（gitignore）

  --ignore=tests/test_e2e_flow.py --ignore=tests/test_perf_api.py

## 4. 作業ルール

### Git ブランチ（ステージ連動）

```
feat/* | fix/* | cursor/*     … 作業ブランチ（必ず dev から切る）
         ↓ merge
       dev                      … 開発統合
         ↓ promotion（Personal 受入後）
    personal                    … 開発者本人の日常運用
         ↓ promotion（Beta 受入後）
      beta                       … 限定公開試用
         ↓ promotion（Production 受入後）
     master                      … 一般公開本番
```

| 操作 | ルール |
|------|--------|
| 日常開発 | `dev` に merge。作業ブランチは `feat/<area>-<topic>` 等 |
| Personal へ | `dev` → `personal` は **Personal 受入目安**（§2）を満たしたタイミングで promotion |
| Beta / master へ | **ユーザー明示指示**があるまで promotion しない |
| hotfix | 運用中ライン（`personal` 等）から `fix/*` を切り、該当ライン + `dev` 両方へ merge |

- UX 改修は **ユーザー明示指示があるまで着手しない**
- **docs に新規ファイルを増やさない**（必要ならユーザー確認）
- 取得仕様変更時は `FETCH_SPEC_VERSION` を bump（`.cursor/rules/fetch-spec-reset.mdc`）
- 話題キーワード調整は `.cursor/rules/topic-keyword-tuning.mdc` の手順に従う
- ステージ判断は `.cursor/rules/release-stages.mdc` を参照
