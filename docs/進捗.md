## 3. 開発環境

### セットアップ

```powershell
git clone https://github.com/soramimi6/LiveChatScope.git
cd LiveChatScope
git checkout dev && git pull origin dev

# Backend
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt -r requirements-dev.txt
uvicorn app.main:app --reload --port 8000

# Frontend（別ターミナル）
cd frontend
copy .env.example .env.local
npm install
npm run dev
```

Linux/macOS: `source backend/.venv/bin/activate`

- Frontend: http://localhost:3000
- API: http://localhost:8000/docs
- DB: `backend/data/livechatscope.db`（gitignore）

### テスト

```bash
# ユニット（54件）
cd backend && . .venv/bin/activate && PYTHONPATH=. pytest tests/ -q \
  --ignore=tests/test_e2e_flow.py --ignore=tests/test_perf_api.py

# フロント
cd frontend && npm run build

# E2E API（Backend 起動済み）
./scripts/e2e-api.sh "https://www.youtube.com/watch?v=8ZaCtuVdWYc"
```

直近: ユニット 54 passed、E2E `8ZaCtuVdWYc` ~1,960 msg PASS、2k 性能 P-02〜P-05 PASS。

## 4. ブランチ・作業ルール

```
master  … リリース
  ↑
  dev   … 日常統合
  ↑
  cursor/* / feat/* / fix/* … 作業ブランチ
```

- 作業ブランチは `dev` から切る。命名: `cursor/<topic>-10cc`
- サブエージェントは commit のみ。マージはマネージャーが `dev` へ
- UX 改修は **ユーザー明示指示があるまで着手しない**
- **docs に新規ファイルを増やさない**（必要ならユーザー確認）

Cursor グローバルルール: `multi-agent-development.mdc`
