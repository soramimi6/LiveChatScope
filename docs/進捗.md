#### #6 反復チューニング手順（人間向け参考）

**なぜ検出されなかったか**

| 要因 | 内容 |
|------|------|
| FE テストなし | `TopicTimelineBar` 等のコンポーネントテスト **0 件**。Recharts 描画は未検証 |
| BE は「理想データ」前提 | `super_chat` テストは DB に `message_type='super_chat'` を **直接 INSERT**。`fetch_worker` → chat-downloader 経路は **未テスト** |
| E2E が疎 | `test_e2e_flow.py` は HTTP 200 と JSON キー存在のみ。スパチャ件数 > 0 や UI 描画は見ない |
| モック fallback | FE は API 失敗時 **mock データ**表示 → ローカルでは「動いて見える」 |
| Phase 5 手動テスト前 | 構成 TL・SC 取得はリリース後の手動テストで初めて顕在化 |

**強化方針（ホワイトリストテスト）**

- **固定フィクスチャ** — `backend/tests/fixtures/` に chat-downloader 出力相当の JSON（text + paid_message + membership_item 含む）を置き、`fetch_worker._normalize_message` と pipeline 全体をホワイトリスト assert
- **FE スナップショット/描画** — `TopicTimelineBar` に blocks 入力 → SVG/bar 要素が存在することを Vitest + RTL で検証
- **E2E 断言の厳格化** — フィクスチャ動画分析後: `super_chat_events.count > 0`、`topic_blocks[].label` の平均長 ≥ N、API レスポンスフィールドの **値域** まで assert
- **モックと実 API の分離** — 手動テスト時は `isMock` バナー確認をチェックリスト必須化

**技術メモ（% 表示 — 却下）**: 全体量が分からないため正確な % は不可。ユーザー判断で #1 却下。

**技術メモ（スーパーチャット — 2026-06-21 調査）**:

| 項目 | 内容 |
|------|------|
| 症状 | DB 上 `uP51LV8d7mw`（53,690 件）は **`text_message` のみ**。`super_chat_*` テーブルも空。収益タブは `none_in_chat` |
| 原因① | `fetch_worker.py` が `ChatDownloader().get_chat(url)` を **引数なし**で呼んでいる。chat-downloader のデフォルト `message_groups=['messages']` は **`text_message` のみ**取得。`paid_message` / `paid_sticker` 等（superchat・tickers グループ）は**最初から除外**される |
| 原因② | chat-downloader の型名は `paid_message` / `paid_sticker` だが、本プロジェクトは `super_chat` / `super_sticker` を前提（`fetch_worker`・Stage3・`super_chat_types`）。**型名マッピング不足** |
| 原因③ | ticker 系・メンバー系は `time_in_seconds` が負になることがあり、Stage0 の `skip_negative_time` で**削除される**可能性 |
| 検証 | テスト動画 `UlemRwXYWHg` で `message_groups=['messages']` → SC 0 件、`['messages','superchat','tickers']` → `paid_message` 等を取得・`money.amount` もパース OK |
| 結論 | **仕様上不可能ではない**。取得パラメータ修正 + 型マッピング + 負時刻扱いの見直しで対応可能。修正不可と判明した場合のみ、収益タブ・Stage3・話題スパチャランキング・export のスパチャ系を除外する方針 |

**技術メモ（チャンネル登録 / メンバー登録 — 2026-06-21 調査）**:

| 種別 | chat-downloader | 状態 |
|------|-----------------|------|
| **メンバーシップ登録** | ✅ `membership_item`, `ticker_sponsor_item` | ✅ **#5 集計対象に追加（確定）** |
| **メンバーシップギフト** | ✅ `sponsorships_gift_purchase_announcement` | ✅ **#5 集計対象に追加（確定）** |
| **チャンネル登録（Subscribe）** | ❌ チャットリプレイに含まれない | 対象外（データ源の制約） |
| 分析アプローチ | メンバー数推移・新規一覧・時間帯別など | 📋 **#8/#4/#2/#7/#3/#6 完了後にブレスト** |

**技術メモ（話題キーワード — 2026-06-21）**: 話題ブロックの表示語は Stage4（Janome 名詞抽出 → `keyword_timeline`）→ Stage5（ブロック代表語）の流れ。現状 `analysis_defaults.json` の **`min_token_length: 2`** により2文字トークンが残りやすい。チューニング候補: `min_token_length` 引き上げ、`stopwords_ja_chat.txt` 拡充、Janome の品詞・複合名詞扱い、Stage5 の代表語選定ロジック、配信ドメイン語の優先/除外。

**技術メモ（話題ブロック一覧 UI — 2026-06-21）**: 現状 `topics-tab.tsx` の `TopicBlocksTable` はサムネイル（`TopicBlockThumbnail`）は非リンク、別列の `JumpLinkButton`（`block.jump_url` = `?t=` 付き YouTube URL）でジャンプ。変更後: サムネイルを `<a href={jump_url}>` でラップ（時刻バッジは維持）、ジャンプ列・`<th>ジャンプ</th>` を削除。BE 変更不要（`jump_url` は API 既存）。

**技術メモ（サマリー構成タイムライン — 2026-06-21）**: `summary-tab.tsx` → `TopicTimelineBar`（Recharts 縦向き stacked `BarChart`）。凡例は Recharts 外の `<ul>` で `blocks` があれば常に表示。グラフは `ResponsiveContainer` 内 — **タブ非表示時の width=0**、長時間配信でのスケール、`stackRow` に `name` 未設定（YAxis `dataKey="name"`）、`h-16` の高さ不足などが描画失敗の候補。話題タブの同一コンポーネントでも再現するか要確認。

## 3. 開発環境

### セットアップ

```powershell
git clone https://github.com/soramimi6/LiveChatScope.git
cd LiveChatScope
git checkout dev && git pull origin dev

# Backend
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt -r requirements-dev.txt
uvicorn app.main:app --reload --port 8000

# Frontend（別ターミナル）
cd frontend
copy .env.example .env.local
npm install
npm run dev
```

Linux/macOS: `source backend/.venv/bin/activate`

- Frontend: http://localhost:3000
- API: http://localhost:8000/docs
- DB: `backend/data/livechatscope.db`（gitignore）

### テスト

```bash
# ユニット（54件）
cd backend && . .venv/bin/activate && PYTHONPATH=. pytest tests/ -q \
  --ignore=tests/test_e2e_flow.py --ignore=tests/test_perf_api.py

# フロント
cd frontend && npm run build

# E2E API（Backend 起動済み）
./scripts/e2e-api.sh "https://www.youtube.com/watch?v=8ZaCtuVdWYc"
```

直近: ユニット 57 passed、E2E `8ZaCtuVdWYc` ~1,960 msg PASS、2k 性能 P-02〜P-05 PASS。
