# LiveChatScope — UX 改修

> 第一弾 POC 完成後の polish / 差別化 backlog  
> **実装ルール**: 明示指示があるまでコード改修しない（UX-21 のみ完了済）

---

## 1. 確定方針

| テーマ | 方針 |
|--------|------|
| **製品姿勢** | 分析・調査ツールとして **正確性最優先**（画面・エクスポート間の一貫性） |
| **フィルター（UX-24）** | **ハイブリッド** — 密度・盛り上がり・Top 投稿者は全件。話題・キーワード・エクスポート messages のみ NG ワード / スタンプ除外 |
| **フィルター再実行** | 変更時は Pipeline **段階 4, 5, 6a, 6b, 7, 8** を連続実行（6c 省略）。「更新中」UI を表示 |
| **エクスポート（UX-23）** | **JSON = 分析結果一式**、**CSV = チャットログのみ**。ExportMenu で役割を説明 |
| **差別化（UX-25）** | 配信者・マネージャー視点。**Must: D2, B1, C1** |
| **ブランド（UX-26）** | 信頼感・製品感（SaaS ツール寄り） |
| **動画リンク（UX-27）** | **進捗画面の動画 ID のみ** YouTube リンク。タイトル行は非リンク |
| **スーパーチャット 0 件（UX-05）** | 取得結果から判別可能なら UI で理由を明示 |

### 未決（実装時に判断）

- UX-23 説明 UI の置き場所（ExportMenu + ツールチップを第一候補）
- UX-05 判別条件の E2E 検証（スーパーチャットあり配信）
