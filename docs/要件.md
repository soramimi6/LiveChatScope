# LiveChatScope — 要件定義

> 主ターゲット: **個人配信者**

## 1. 目的

YouTube ライブ配信の**チャットリプレイ**を取得し、**配信後の振り返り・次回改善**に使える分析と、任意時刻へのジャンプを提供する。

### プロダクト定位

> LiveChatScope = チャットをダウンロードするツールではなく、**配信後のチャットを配信者の振り返り資料に変える**ツール

YouTube Analytics の代替ではない。**チャットデータに基づく運営振り返り**に特化する。

## 2. ターゲットユーザー

### 主ターゲット: 個人配信者

- 配信終了後に「どこが盛り上がったか」「どのコーナーが伸びなかったか」を知りたい
- スパチャのお礼・切り抜き候補の抽出など、**配信後の実務**を効率化したい
- 高度な分析基盤（MCN・事務所向けダッシュボード等）は持っていない

### 想定しない主ターゲット

- 企業・組織・MCN 所属で、既に専用分析ツール・運営体制を持つ層  
  （将来 Phase D で一般公開しても、**機能設計の主眼は個人配信者**とする）

### 副次ユーザー

- 個人配信者の協力者（編集者・モデレーター）— エクスポート・ジャンプリンクの共有利用

## 3. スコープ

| 対象 | 含む | 含まない |
|------|------|----------|
| データ | ライブチャットリプレイ | 動画下の通常コメント欄 |
| タイミング | 配信終了後（VOD） | 配信中リアルタイム（Phase A では後回し可） |
| 分析 | 配信者向け振り返り・話題俯瞰 | 同時視聴者数・登録者増等（Analytics 領域） |
| UI | 分析・リンク・エクスポート | 動画プレイヤー埋め込み |

### 分析上の前提・限界（UI で明示する）

- チャットは**視聴者の反応**であり、配信者の発言内容そのものではない
- 話題ラベルは「配信内容」ではなく**チャット上の話題区間**として表示する
- 盛り上がりスパイクはネガティブ反応（炎上等）も含む場合がある
- チャットリプレイが無効な配信は分析対象外

## 4. 機能要件

### FR-1 取得

- [ ] YouTube URL（`watch?v=` / `youtu.be`）を入力できる
- [ ] ライブチャットリプレイを非同期で取得する
- [ ] 進捗（件数・状態）を表示する
- [ ] 取得完了後、DB またはストレージに保存する
- [ ] リプレイ無効・非公開・取得失敗時に理由を表示する

### FR-2 閲覧・検索（補助機能）

一般視聴者向けの「ネタ探し」より、配信者の深掘り調査用として位置づける。

- [ ] 時刻順でメッセージ一覧を表示する（ページング）
- [ ] キーワード検索（部分一致）
- [ ] 投稿者名・メッセージ種別でフィルタ

### FR-3 分析 — 基本（MVP 基盤）

- [ ] コメント密度タイムライン（1分/5分単位）
- [ ] 投稿者 Top N（件数）
- [ ] メッセージ種別集計（text / super_chat 等）
- [ ] スパチャ集計（合計・件数・時系列、取得データに含まれる場合）

### FR-3 分析 — 差別化コア（Phase A+）

配信全体を俯瞰する分析。実装依存関係: 形態素解析 → キーワード → 話題ブロック → 遷移・アクティブユーザ。

| ID | 機能 | 内容 |
|----|------|------|
| FR-3a | 盛り上がり候補 | 密度スパイク Top N を抽出し、ジャンプ URL 一覧化 |
| FR-3b | キーワード分析 | 日本語形態素解析＋チャット向けストップワード。配信全体・時間帯別の語頻度 |
| FR-3c | 話題ブロック分割 | 時系列ウィンドウ＋語分布変化点で区間分割。各区間に代表語ラベル・件数・ジャンプ URL |
| FR-3d | 話題遷移 | 話題ブロック間の遷移を可視化（帯グラフ・遷移一覧等） |
| FR-3e | 話題別アクティブユーザ | 各話題ブロック内の Top 投稿者・ユニークユーザ数 |

- [ ] FR-3a 盛り上がり候補
- [ ] FR-3b キーワード分析
- [ ] FR-3c 話題ブロック分割
- [ ] FR-3d 話題遷移
- [ ] FR-3e 話題別アクティブユーザ

### FR-3 分析 — 配信者向け（Phase A+）

| ID | 機能 | 内容 |
|----|------|------|
| FR-3p1 | スパチャお礼リスト | 時刻・金額・投稿者・メッセージ・ジャンプ URL。CSV / Markdown 出力 |
| FR-3p2 | 切り抜き候補パック | 盛り上がり区間 ±30秒 を YouTube 説明欄用タイムスタンプ形式で出力 |
| FR-3p3 | 低活動区間 | 密度が平均以下の連続区間を表示（間延び・離脱の疑い） |
| FR-3p4 | 常連コア層 | 配信全体または話題別に、継続的に発言したユーザの可視化 |
| FR-3p5 | スパチャ×密度 | 同一タイムライン上にコメント密度とスパチャを重ね表示 |

- [ ] FR-3p1 スパチャお礼リスト
- [ ] FR-3p2 切り抜き候補パック
- [ ] FR-3p3 低活動区間
- [ ] FR-3p4 常連コア層
- [ ] FR-3p5 スパチャ×密度

### FR-3 分析 — 振り返りサマリー（Phase A+）

個人配信者が配信直後に一覧で把握するためのダッシュボード。企業向けレポート機能はスコープ外。

| ID | 機能 | 内容 |
|----|------|------|
| FR-3s1 | 配信サマリー | 総コメント数、ピーク時刻、スパチャ合計、話題ブロック数、Top 盛り上がり（1画面） |
| FR-3s2 | 話題別スコアカード | 各区間: コメント数、ユニークユーザ、スパチャ、代表語、ジャンプ URL |
| FR-3s3 | 構成タイムライン | 話題ブロックを横棒で俯瞰（「どこで何が起きたか」） |

- [ ] FR-3s1 配信サマリー
- [ ] FR-3s2 話題別スコアカード
- [ ] FR-3s3 構成タイムライン

### FR-4 ジャンプ

- [ ] 各メッセージに `watch?v=...&t=...s` リンクを付与
- [ ] 検索結果・盛り上がり候補・話題ブロック・スパチャ一覧からも同リンクを生成

### FR-5 エクスポート

- [ ] JSON エクスポート（生データ）
- [ ] CSV エクスポート（分析列含む）
- [ ] 振り返りサマリー・切り抜き候補・スパチャお礼リストの Markdown 出力（Phase A+）

#### Phase D 認証・データ境界（確定 2026-06-21）

> 詳細設計は [アーキテクチャ.md](アーキテクチャ.md) / [API仕様.md](API仕様.md) 実装時に追記。UX-20（設定永続化）は D3 で実装。

**目的**

| # | 目的 |
|---|------|
| 1 | UX-20: ユーザー設定（NG ワード等）の永続化 |
| 2 | 悪用防止（無制限 URL 投入の抑止） |
| 3 | 分析結果の所有者境界（他人のデータを見せない） |

**認証（D1）**

| 項目 | 仕様 |
|------|------|
| ログイン手段 | **Google OAuth のみ**（v1） |
| セッション | **HttpOnly Cookie** + サーバー側 `sessions` テーブル |
| ログイン必須範囲 | **分析開始（POST `/videos`）以降すべて**。LP のみ未ログイン可 |
| ローカル開発 | 環境変数 **`AUTH_DISABLED=true`** で認証スキップ可 |

**データ所有権（D2）**

| データ | 所有者 | 他人からの閲覧 |
|--------|--------|----------------|
| 分析ジョブ（`videos` および派生テーブル） | `owner_user_id` | **不可（403）** |
| ユーザー設定（`user_settings`） | 本人のみ | 不可 |
| `messages`（取得キャッシュ） | 動画 ID 単位（所有者なし） | D5 まで各自 fetch |

- スキーマ: `videos` に `owner_user_id` を追加、または `analysis_jobs` へ分離
- 一意制約: **`(video_id, owner_user_id)`** — 1 ユーザー 1 動画 1 ジョブ
- **同一ユーザー・同一動画の再分析**: 既存ジョブを **上書き**（リセットして再取得・再分析）
- 既存 POC データ: 本番公開前に DB purge 推奨（`owner_user_id` なし行は移行しない）

**設定永続化 UX-20（D3）**

```text
user_settings（デフォルト）
  → 分析開始時に videos.display_filter_json へコピー
  → 以降は動画単位で refilter（現行 Q1/Q7/Q8 と同じ）
```

**レート制限（D4）**

| 対象 | 上限 |
|------|------|
| 新規分析開始 | **3 本 / 日 / ユーザー** |
| 同時実行中ジョブ | **1 本 / ユーザー** |
| 超過時 | `429 Too Many Requests` + 日本語メッセージ |

**アカウント削除（D4）**

- ユーザー・セッション・所有ジョブ・ユーザー設定を **物理削除（CASCADE）**

**messages 共有キャッシュ（D5 — v1 見送り）**

- v1: ユーザーごとに **フル fetch**
- D5 以降: `FETCH_SPEC_VERSION` 一致時、他ユーザーの `messages` を流用して fetch をスキップ（分析はユーザーごとに実行）

**実装フェーズ**

| Phase | 内容 |
|-------|------|
| D1 | Google OAuth + sessions + ミドルウェア + `AUTH_DISABLED` |
| D2 | `owner_user_id` + 認可 + 上書き再分析 |
| D3 | `user_settings` + 分析開始時コピー（UX-20） |
| D4 | レート制限 + 利用規約・プライバシー + 削除 API |
| D5 | messages 共有キャッシュ |

**法務（D4 最低限）**

- 利用規約（非公式取得・自己責任・禁止事項）
- プライバシーポリシー（email、分析履歴、Cookie）
- 13 歳未満は対象外

**却下・見送り（関連 UX）**

| ID | 内容 |
|----|------|
| UX-03 | 取得件数更新頻度の改善 — **却下** |
| UX-14 | 低活動区間の相対判定アルゴリズム — **却下**（赤帯表示は実装済） |

## 7. 非機能要件（目安）

| 項目 | 目標 |
|------|------|
| 規模 | 10万メッセージ/配信を想定 |
| 検索 | インデックス付き DB で数秒以内 |
| 可用性 | chat-downloader バージョン固定・定期動作確認 |
| セキュリティ | API キー等をフロントに露出しない |
| 表現 | 話題ラベル等に「推定」「チャット上の話題」と明記 |

## 8. データ項目（正規化後・最小）

```
videos:
  video_id, title, channel, duration, fetched_at, message_count

messages:
  message_id, author_id, author_name, message_type, text,
  time_in_seconds, timestamp,
  super_chat_amount, super_chat_currency (nullable)

analysis (派生・Phase A+):
  topic_blocks: block_id, video_id, start_sec, end_sec, label, message_count, unique_authors
  highlights: video_id, time_in_seconds, score, rank
```

## 9. 成功基準

### Phase A

- チャットリプレイ有効な配信で安定取得できる
- 密度グラフ・ジャンプリンク・基本検索が期待どおり動作する
- 10万コメント級で UI が実用速度

### Phase A+

- 個人配信者が配信サマリー1画面で振り返りの起点を得られる
- 話題ブロック＋ジャンプ URL で「どの区間を見返すか」が判断できる
- スパチャお礼リスト・切り抜き候補をエクスポートして配信後作業に使える
